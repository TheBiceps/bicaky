This skill guides creation of clean, modern, and visually polished frontend interfaces. Every output should be production-ready code with attention to detail in layout, typography, color, and motion.

The user provides a frontend request: a component, page, section, or full application to build. They may include context about purpose, audience, brand, or technical requirements.

## Design Approach

Before writing code, consider the context:
- **Purpose**: What does this interface do? Who is it for?
- **Aesthetic**: Choose a clear visual direction — minimal, bold, playful, corporate, editorial, etc.
- **Consistency**: Maintain a cohesive look throughout with unified spacing, colors, and typography.

Then produce working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Functional and production-ready
- Visually clean and intentionally designed
- Responsive across screen sizes
- Accessible with proper semantic HTML

## Frontend Guidelines

### Typography
- Use distinctive, well-paired fonts from Google Fonts or system fonts
- Establish a clear type hierarchy: headings, body, captions
- Use proper line-height, letter-spacing, and font-weight for readability

### Color
- Define a cohesive color palette using CSS variables
- Use a dominant color with intentional accent colors
- Ensure sufficient contrast for readability (WCAG AA minimum)
- Support both light and dark themes when appropriate

### Layout
- Use CSS Grid and Flexbox for responsive, structured layouts
- Apply generous and consistent spacing
- Create visual rhythm with alignment and whitespace
- Design mobile-first, then enhance for larger screens

### Motion & Interaction
- Add subtle animations for page load, scroll reveals, and hover states
- Use CSS transitions for simple effects; GSAP or Framer Motion for complex ones
- Keep animations purposeful — enhance usability, don't distract
- Respect `prefers-reduced-motion` for accessibility

### Visual Details
- Use backgrounds with depth: gradients, subtle patterns, or blur effects
- Apply consistent border-radius, shadows, and border treatments
- Add micro-interactions on buttons, cards, and form elements
- Use SVG icons over emoji for a professional look

## Code Quality
- Write semantic HTML with proper heading structure
- Use CSS custom properties for theming
- Keep JavaScript minimal and purposeful
- Comment only where logic isn't self-evident
- Optimize for performance: minimal DOM, efficient selectors, lazy loading where needed
